# Finetuning Plan: Hinglish Chatty Friend on Llama-3.1-8B-Instruct

## Context
**Goal:** Finetune Llama-3.1-8B-Instruct into a chatty Indian friend that speaks Hinglish, using the [Hinglish-Everyday-Conversations-1M](https://huggingface.co/datasets/Abhishekcr448/Hinglish-Everyday-Conversations-1M) dataset. The model should primarily be a Hinglish chatbot (reasoning preservation is not a priority).

**Hardware:** RunPod — 1x NVIDIA L4 (24GB VRAM), 128-core AMD EPYC, ~512GB RAM, storage expandable.

**Access:** Claude Code installed directly on the RunPod VM for full control.

**Dataset:** 1M synthetic Hinglish conversation pairs (input/output), generated by GPT-4o-mini. Covers everyday topics — food, traffic, office, movies, etc. MIT license.

---

## Strategy: QLoRA on Single L4 GPU

**Why QLoRA (not full finetune or standard LoRA):**
- Llama-3.1-8B in 4-bit = ~5GB VRAM for weights
- LoRA adapters add ~100-200MB
- With gradient checkpointing, total VRAM usage ~12-16GB — well within 24GB
- No multi-GPU complexity needed
- Quality is excellent for style/tone transfer tasks — we're teaching conversational style, not new knowledge

**No DeepSpeed needed** — single GPU, QLoRA keeps everything comfortably in 24GB.

## Implementation Steps

### Step 0: Prep
- Expand RunPod storage to **100-150GB** (model bf16 for merge = 16GB, dataset ~1GB, checkpoints, merged output)
- Install Claude Code on the VM
- `huggingface-cli login` (Llama-3.1 is gated — accept license on HF first)

### Step 1: Install Dependencies
```bash
pip install -r requirements.txt
```

### Step 2: Preprocess Data (`preprocess.py`)
- Downloads the HF dataset
- Filters out responses <10 chars or >500 chars
- Takes 300K sample subset (enough for strong style transfer)
- Converts to Llama-3.1 chat format with Hinglish system prompt
- Splits 95% train / 5% eval
- Saves to `./processed-data/`

```bash
python preprocess.py --num_samples 300000
```

**Chat format used:**
```
<|begin_of_text|><|start_header_id|>system<|end_header_id|>

Tu ek friendly Indian dost hai jo Hinglish mein baat karta hai. Casual, fun, aur relatable tone mein reply kar. Short aur natural responses de, jaise koi close friend baat kar raha ho.<|eot_id|>
<|start_header_id|>user<|end_header_id|>

{input}<|eot_id|>
<|start_header_id|>assistant<|end_header_id|>

{output}<|eot_id|>
```

### Step 3: Train (`train.py`)
**QLoRA config:**
- 4-bit NF4 quantization with double quantization
- LoRA rank=64, alpha=128, dropout=0.05
- Target all linear layers: q/k/v/o_proj, gate/up/down_proj

**Training hyperparameters:**
- Epochs: 3
- Batch size: 4, gradient accumulation 4 → effective batch size 16
- Learning rate: 2e-4, cosine schedule, 5% warmup
- bf16, gradient checkpointing ON
- Optimizer: paged_adamw_8bit
- Save every 1000 steps, keep best 3 by eval loss

```bash
python train.py
# Optional: python train.py --use_wandb
```

### Step 4: Test (`inference.py`)
```bash
# Test with LoRA adapter
python inference.py --model_path ./hinglish-friend-checkpoints/final --use_adapter

# Interactive chat
python inference.py --model_path ./hinglish-friend-checkpoints/final --use_adapter --interactive
```

### Step 5: Merge & Export (`merge.py`)
```bash
python merge.py
```
Merges LoRA into base model → `./hinglish-friend-merged/`

### Step 6: Test Merged Model
```bash
python inference.py --model_path ./hinglish-friend-merged --interactive
```
